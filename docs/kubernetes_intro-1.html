‚ùå Problem 2: No Auto-healing

Scenario: Container gets killed

Current State: Application becomes inaccessible

Required Action: Manual intervention by DevOps engineer

# Manual check required
docker ps

# Manual restart required

Problem: In production with 10,000+ containers, manual monitoring is impossible

‚úÖ Solution: Automatic Rollout

Kubernetes Auto-healing Process:

Result: New container starts even before old one dies completely

‚úÖ Solution: Replica Sets & HPA

Manual Scaling with YAML:
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 10  # Scale from 1 to 10
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app
        image: my-app:latest
Automatic Scaling with HPA:
apiVersion: autoscaling/v2
kind: HorizontalPodAutoscaler
metadata:
  name: my-app-hpa
spec:
  scaleTargetRef:
    apiVersion: apps/v1
    kind: Deployment
    name: my-app
  minReplicas: 1
  maxReplicas: 100
  metrics:
  - type: Resource
    resource:
      name: cpu
      target:
        type: Utilization
        averageUtilization: 80

Result: Automatic scaling based on CPU/memory thresholds

üèóÔ∏è Basic Deployment Example

apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
  labels:
    app: nginx
spec:
  replicas: 3
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx
        image: nginx:1.21
        ports:
        - containerPort: 80
        resources:
          requests:
            memory: "128Mi"
            cpu: "100m"
          limits:
            memory: "256Mi"
            cpu: "200m"

üåê Service Configuration

apiVersion: v1
kind: Service
metadata:
  name: nginx-service
spec:
  selector:
    app: nginx
  ports:
    - protocol: TCP
      port: 80
      targetPort: 80
  type: LoadBalancer

üîß Essential kubectl Commands

# Create deployment
kubectl apply -f deployment.yaml

# Get all pods
kubectl get pods

# Get all services
kubectl get services

# Get deployment status
kubectl get deployments

# Scale deployment
kubectl scale deployment nginx-deployment --replicas=5

# Get pod logs
kubectl logs pod-name

# Describe pod details
kubectl describe pod pod-name

# Execute command in pod
kubectl exec -it pod-name -- /bin/bash

# Delete deployment
kubectl delete deployment nginx-deployment

# Get cluster info
kubectl cluster-info

# Get nodes
kubectl get nodes
